Highlight #1 Version Control

Version Control is a way of having multiple copies of a project throughout the working phase that the user may go back to at any time. With this, if the user decided they wanted to work on a new version of the project, but still have the older version, they would have both options. There are three types of Version Control Systems or VCS: local, centralized, and distributed. Local VCS is the file the user can save on the computer, and only that computer has access to it. The Centralized VCS can be used by multiple computers, and stores files within a general server. This is a great method for doing shared work with team members. Lastly, the Distributed VCS, similar to the Centralized VCS, can be used to work from multiple computers by mirroring the repository onto computer with access to the server. This is important because it allows us to work on the same content while in different locations. This is relevant to this course because we will be using Github, which is a remote repository.

Highlight #2 History of Git

Git was established from the flames of a broken bond. An open source software named Linux kernel used to partner with a Distributed VCS named BitKeeper. When the relationship went south, Linux had to come up with its own form of a VCS. Luckily, since they were using BitKeeper for long, they were able to create their own. This is what we know as Git. This is important because it is a fast, efficient way to work on large projects. We need to use systems like Git to work on projects in class.

Highlight #3 Git Basics

Git is similar to a mini file system in a way that it takes snapshots of previous work and stores it as a reference. This is different from VCS that saves the whole edited version of a file. Git mostly reads local files. This means if the user needed from their local computer, Git can find it easily. The best part about Git is that the data only increases. There are three stages of a Git: the working directory, staging area, and repository. Git is important because we will be able to access, and add data without the fear of anything being deleted.

Highlight #4 Getting a Git Repository

Initializing a repository is different depending on the system the user is using. For Mac, the user would type in “$ cd /Users/user/your_repository” into the terminal, then type “$ git init” (this creates a subdirectory of the name “.git”). There are many different functions that the user may use to manipulate the Git. The “$ git add .” command adds all the files to the Git. Then to commit the files, the “$ git commit –m” command is used with a descriptions in quotations. This is important because we will be using these commands in class to add and commit these files into our Github accounts.

Highlight #5 Recording Changes to the Repository

There are many different commands that may be used in the terminal to manipulate files. Some of the most common ones are “git status,” “.gitignore,” and “git diff.” The “git status” command checks the status of a file after it is modified. The “.gitignore” command allows the user to ignore files. Lastly, the “git diff” command allows the user see the lines that were added or removed. These commands are important because they will be used when working with Github. We will use them when editing files and it is a great safety net if, God forbid, anything happens to one of our files.
